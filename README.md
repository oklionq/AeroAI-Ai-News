# AI News Agent

Автономный Telegram-бот, который собирает новости про ИИ из RSS, Reddit и Hacker News, фильтрует их с помощью LLM на предмет важности, генерирует короткие посты и отправляет администратору на подтверждение перед публикацией.

## Локальный запуск

1. Установите Python 3.11+
2. Создайте виртуальное окружение: `python -m venv venv` и активируйте его (Windows: `venv\Scripts\activate`, Mac/Linux: `source venv/bin/activate`)
3. Установите зависимости: `pip install -r requirements.txt`
4. Скопируйте `.env.example` в `.env` и заполните все обязательные переменные (см. ниже).
5. Запустите бота: `python -m bot.main`

База данных SQLite автоматически создастся в папке `data/bot.db` (если не указана другая в `DATABASE_URL`).

## Деплой на Railway

1. Создайте проект на Railway (railway.app).
2. Подключите репозиторий GitHub.
3. Добавьте сервис. При необходимости укажите тип сервиса как **Worker** (или просто оставьте, так как Railway использует `Procfile` `worker: python -m bot.main`).
4. Настройте переменные окружения в панели Railway (раздел Variables).
5. Добавьте Persistent Volume для базы данных (если не используете Postgres) и смонтируйте его, например, в `/app/data` (или обновите `DATABASE_URL` до пути в volume).

## Переменные окружения (.env)

- `TELEGRAM_BOT_TOKEN`: токен бота от @BotFather.
- `ADMIN_CHAT_ID`: ваш ID для получения на модерацию (можно узнать у @userinfobot).
- `TARGET_GROUP_ID`: ID вашей группы для публикации новостей (с минусом, например, `-100...`).
- `TOPIC_MAPPING`: JSON, сопоставляющий категории (model_release, feature_update, benchmark, pricing, org_news, other) с ID топиков в группе (message_thread_id). Узнать thread_id можно, кликнув правой кнопкой на сообщение в теме и скопировав ссылку (цифра на конце — ID темы).
- `OPENAI_API_KEY`: API-ключ OpenAI.
- `FILTER_MODEL` / `GENERATION_MODEL`: модели OpenAI (по умолчанию `gpt-4o-mini`).
- `MAX_BUDGET_USD`: Лимит трат в USD. При достижении бот ставится на паузу.
- `POLL_INTERVAL_MINUTES`: интервал опроса источников в минутах.
- `DATABASE_URL`: Строка подключения к БД. Оставьте пустым для SQLite.

## Управление источниками

Список источников находится в базе данных в таблице `sources`. 
Чтобы добавить новый источник или выключить существующий, можно отредактировать эту таблицу (например, используя утилиту `sqlite3` или подключившись к Postgres).
По умолчанию источники загружаются автоматически при первом запуске бота, если таблица пуста.
